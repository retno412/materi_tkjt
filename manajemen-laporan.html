💼 PROFESSIONAL WRITING STANDARDS

📝 Manajemen Laporan & Penulisan Dokumen Teknis

Kerapian tulisan mencerminkan kedalaman isi pikiran. Pelajari standarisasi format dokumen ilmiah dan industri software.

📏 Mengapa Format Laporan Harus Teratur?

Di industri teknologi, dokumentasi tertulis adalah segalanya. Mulai dari berkas Software Requirement Specification (SRS) hingga berkas laporan proyek bulanan. Jika tulisan Anda berantakan, pembaca (atasan atau klien) akan menganggap kualitas sistem/aplikasi yang Anda bangun juga berantakan.

📋 Golden Rules Penulisan Dokumen (Wajib Diikuti)

1. Aturan Tipografi Standar Karya Ilmiah
  • Jenis Huruf: Times New Roman (Ukuran 12) atau Arial/Calibri (Ukuran 11). Jangan pernah mencampur lebih dari 2 jenis font dalam satu dokumen!
  • Jarak Spasi: Gunakan spasi 1.5 untuk kenyamanan membaca laporan panjang.
  • Rata Kanan Kiri (Justify): Selalu pastikan paragraf teks menggunakan format rata kanan-kiri (Ctrl + J), bukan rata kiri acak-acakan.
2. Standardisasi Penulisan Istilah Asing (Koding)

Karena jurusan kita erat dengan istilah berbahasa Inggris, semua kosakata asing wajib ditulis menggunakan gaya Miring (Italic).

  • Salah: Kami melakukan proses debugging pada fungsi looping web server.
  • Benar: Kami melakukan proses debugging pada fungsi looping web server.
3. Tata Cara Penulisan Kode Program di Laporan

Jangan menempelkan cuplikan screenshot kode program dalam format gambar buram. Gunakan elemen tabel satu kolom berlatar belakang abu-abu tipis dengan jenis font berjarak serasi (Courier New atau Consolas) agar teks kode program dapat disalin untuk diaudit oleh guru.
